# GPT Desktop

`GPT Desktop` — это лёгкая desktop-обёртка над `chatgpt.com` на Python с использованием `PySide6` и `Qt WebEngine`.

Приложение открывает ChatGPT в отдельном окне с кастомной шапкой, полупрозрачным оформлением и сохранением сессии между перезапусками.

## Что умеет

- открывает `https://chatgpt.com` в отдельном desktop-окне;
- использует собственную шапку окна без системной рамки;
- поддерживает перемещение, изменение размера, сворачивание, разворачивание и режим `Always on top`;
- применяет кастомные стили поверх веб-интерфейса;
- сохраняет cookies и локальное хранилище, поэтому повторный вход в аккаунт при каждом запуске не требуется.

## Требования

- Linux;
- Python 3.10+;
- установленный `pip`.

## Быстрый запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python main.py
```

Также можно запускать через скрипт:

```bash
./run.sh
```

Скрипт сам попытается использовать `venv`, `.venv`, а если окружение не найдено — системный `python3`.

## Установка ярлыка приложения

```bash
chmod +x install_linux_desktop.sh
./install_linux_desktop.sh
```

Скрипт:

- копирует `.desktop`-файл в `~/.local/share/applications`;
- устанавливает иконку в `~/.local/share/icons/hicolor/scalable/apps`;
- обновляет путь до текущей директории проекта;
- делает ярлык исполняемым.

После этого приложение можно запускать из меню приложений как обычную desktop-программу.

## Структура проекта

- `main.py` — точка входа;
- `app/main.py` — основное окно и встраивание `Qt WebEngine`;
- `app/injected_style.py` — CSS/JS, которые внедряются в страницу;
- `run.sh` — удобный запуск проекта;
- `install_linux_desktop.sh` — установка desktop-ярлыка;
- `assets/` — иконки и статические файлы.

## Где хранится сессия

Данные браузерного профиля, cookies и local storage сохраняются в стандартной директории приложения `Qt` (`AppDataLocation`). Благодаря этому авторизация сохраняется между перезапусками.

## Ограничения

- приложение зависит от текущей разметки `chatgpt.com`, поэтому после крупных изменений сайта стили могут потребовать правок;
- прозрачность и blur зависят от оконного менеджера и композитора в системе;
- проект не является официальным клиентом OpenAI.
